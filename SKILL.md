# Character Analysis（人物深度分析）

根据用户提供的**一段或多段文本**，深入分析其中指定的**人物**（真实人物或虚构角色均可），产出一份**有原文证据支撑**的 Markdown 分析报告。

## 核心原则（必须遵守）

1. **证据优先，禁止臆造**：每一条性格、动机、行为模式的结论，**必须**引用原文片段作为证据。无法在文本中找到依据的，不要写；可以在"信息不足"一节中标注。
2. **区分"文本明示"与"合理推断"**：直接写明的事实标为「明示」，需要读者推理得出的标为「推断」，并说明推断链条。
3. **忠于文本，不带入外部知识**：即使是真实历史人物，也只基于用户给的文本下结论。若文本与你已知的常识冲突，以文本为准，并可在末尾"备注"中提示冲突。
4. **多文本整合**：当提供多段/多个文本时，跨文本整合同一人物的信息，并在证据中标注来源（如「文本2」「第3章」或文件名）。

## 工作流程

### 第 1 步：确认分析对象与输入
- 明确要分析的**人物名称**（用户已指定则直接用；未指定且文本只有一个主要人物则默认分析它；有多个且未指定时，先列出候选人物让用户选）。
- 收集所有输入文本。多文件时为每段标注**来源标识**（文件名或编号），后续引用要带上。

### 第 2 步：通读 + 提取证据
逐文本通读，为下列每个维度**先收集原文片段**，再下结论。先证据后结论，不要反过来。

### 第 3 步：按模板产出报告
使用下方"报告模板"。每个维度下，结论后紧跟 `> 证据：` 引用。

### 第 4 步：自检
产出后核对：
- 每条结论是否都有证据？没有的删掉或移到"信息不足"。
- 是否混入了文本里没有的外部知识？有则删除或移到"备注"。
- 引用是否标了来源（多文本时）？

## 报告模板

```markdown
# 人物分析：{{人物名称}}

> 分析基于：{{文本来源说明，如 "文件 a.txt、b.md" 或 "用户提供的 3 段文本"}}

## 一、基本信息与关系网
- **身份/角色**：……
  > 证据：「原文引用」（来源）
- **关系网**：与 X 是……，与 Y 是……
  > 证据：「原文引用」（来源）

## 二、性格特质
| 特质 | 类型 | 证据 |
|------|------|------|
| 例：果断 | 明示/推断 | 「原文引用」（来源） |

## 三、动机与目标
- **核心动机**：……（明示/推断）
  > 证据：「原文引用」（来源）
- **目标**：……
  > 证据：「原文引用」（来源）

## 四、行为模式与心理变化弧线
- **典型行为模式**：……
  > 证据：「原文引用」（来源）
- **变化弧线**：起点 → 转变 → 终点（若文本体现了变化）
  > 证据：「原文引用」（来源）

## 五、关键事件与转折点
1. **{{事件}}**：对人物的影响……
   > 证据：「原文引用」（来源）

## 六、与其他人物的关系
- **{{对方}}**：关系性质 / 互动特点……
  > 证据：「原文引用」（来源）

## 七、信息不足 / 无法确定
- 文本未提供关于……的信息，因此本报告未涵盖该维度。

## 备注（可选）
- 文本与外部常识的冲突、需要用户澄清的点等。
```

## 输出约定
- 报告语言与输入文本/用户语言保持一致（用户用中文则用中文）。
- 默认将报告直接输出在对话中；若用户要求保存，则写入文件（如 `character-analysis-{{人物名}}.md`）。
- 引用原文时尽量精简（截取关键句），避免大段照搬。
